# autodoc-mvp

Пайплайн для автоматической генерации и обновления спецификаций программных компонентов в формате FODT (LibreOffice) из Python-кода.

Извлекает структуру кода через AST, генерирует текстовые описания через LLM и заполняет корпоративный шаблон ЕСПД. При повторных запусках обновляет только изменившиеся секции — без полной перегенерации.

## Требования

- Python 3.9+
- API-ключ [OpenRouter](https://openrouter.ai)
- LibreOffice для просмотра результата

## Установка

```bash
git clone https://github.com/milamorozova/autodoc-mvp-test
cd autodoc-mvp-test
pip install certifi
export OPENROUTER_API_KEY=ваш_ключ
```

Установить git hook для автоматического запуска при коммите:

```bash
chmod +x .git/hooks/post-commit
```

## Использование

Положите исходный Python файл в `work/`, затем запустите генерацию:

```bash
# Первый запуск — полная генерация
python3 -m tools.gen_spec_md --mode clean --enrich \
    --component-name "MyComponent" \
    --component-type APPLICATION \
    --fodt templates/RU_ECO_00101-01_90__1_.fodt

# Обновление после изменений в коде
python3 -m tools.gen_spec_md --mode update --enrich \
    --component-name "MyComponent" \
    --component-type APPLICATION \
    --fodt templates/RU_ECO_00101-01_90__1_.fodt
```

Если настроен git hook — update запускается автоматически при каждом коммите затрагивающем `work/`.

Результат сохраняется в `docs/spec_MyComponent.fodt`.

## Параметры

| Параметр | Описание |
|---|---|
| `--mode clean` | Полная генерация с нуля |
| `--mode update` | Обновление только изменившихся секций |
| `--mode auto` | Автовыбор режима |
| `--enrich` | Включить обогащение через LLM |
| `--component-name` | Имя компонента |
| `--component-type` | Тип: APPLICATION, MODULE, COMPONENT, LIBRARY |
| `--fodt` | Путь к FODT шаблону |
| `--cid` | CID компонента (если есть) |
| `--espd-code` | Код ЕСПД |

## Структура проекта

```
tools/              — основная логика пайплайна
tools/extractors/   — экстракторы AST по языкам
work/               — исходные файлы для документирования
docs/               — сгенерированные спецификации (.fodt, .md)
snapshots/          — JSON снапшоты AST для diff
templates/          — FODT шаблон LibreOffice
registry.json       — хэши документов для отслеживания ручных правок
```

## Как это работает

**clean** — строит AST, отправляет все сущности в LLM, заполняет шаблон целиком.

**update** — читает хэш коммита из метаданных FODT, делает `git diff` с текущим HEAD, находит затронутые публичные сущности по номерам строк, обогащает только их через LLM, патчит соответствующие секции документа. Остальные секции не трогает — в том числе ручные правки пользователя.

При обнаружении ручных правок в FODT система сохраняет `.bak` копию перед обновлением.

## Ограничения

- Поддерживается только Python (экстрактор для C в разработке)
- FODT шаблон привязан к конкретной XML структуре — изменение шаблона требует обновления якорей в `fodt_filler.py`
- LLM не всегда возвращает ответ в ожидаемом формате — парсер обрабатывает несколько форматов, но полная устойчивость не гарантируется
- Определение изменений тела функции основано на сравнении диапазона строк, изменения логики без изменения длины функции не обнаруживаются
